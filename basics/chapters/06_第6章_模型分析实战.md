# 第 6 章 模型分析实战

## 6.1 DeepSeek V4-Pro 架构分析

### 6.1.1 总体架构概览（输入层→隐藏层→输出层）

DeepSeek V4-Pro 是深度求索（DeepSeek）于 2025 年发布的旗舰级大语言模型，采用 MoE（Mixture-of-Experts）稀疏激活架构，总参数量约 1 万亿（1T），但每次推理仅激活约 250B 参数。以下从输入层、隐藏层、输出层三个维度拆解其完整网络结构。

| 网络层次 | 组件名称 | 具体配置 | 功能说明 |
| --- | --- | --- | --- |
| 输入层 | Token Embedding | 词表 ~128K，嵌入维度 7168 | 将 Token ID 映射为稠密向量 |
| 输入层 | Position Encoding | RoPE 旋转位置编码 | 注入相对位置信息，支持外推至 128K-1M |
| 隐藏层 | Transformer Block × N | 约 60-80 层 Decoder-Only | Pre-RMSNorm → MLA → Residual → Pre-RMSNorm → MoE FFN → Residual |
| 隐藏层 | MLA Attention | Q 头 H=128，潜在维度 d_lv=512 | K/V 低秩投影压缩，KV Cache 降 10-15x |
| 隐藏层 | MoE FFN | 256+ 细粒度专家 + 2 共享专家, Top-8~12 | SwiGLU 门控 FFN，稀疏激活 |
| 隐藏层 | Pre-RMSNorm | 每子层前归一化 | 相比 LayerNorm 省去均值计算，减 15% 开销 |
| 隐藏层 | Residual 残差连接 | Pre-Norm: x + Sublayer(Norm(x)) | 稳定深层网络训练 |
| 输出层 | Final RMSNorm | 最后一层归一化 | 稳定输出分布 |
| 输出层 | LM Head | d_model → 词表 128K | 线性投影输出 logits，可选权重绑定 |

从宏观视角看，DeepSeek V4-Pro 的完整数据流为：输入 Token → Embedding + RoPE → N 层 Transformer Block（每层执行 MLA 注意力 + MoE FFN）→ Final RMSNorm → LM Head → 输出 logits。与传统 Dense 模型最大的不同在于其 FFN 层采用了 MoE 架构，每层 FFN 并非单一网络而是数百个专家网络的组合。

### 6.1.2 MoE 混合专家架构

MoE（Mixture-of-Experts）是 DeepSeek V4-Pro 最核心的架构创新。与传统的密集型（Dense）Transformer 不同，MoE 将每一层的 FFN（前馈神经网络）拆分为多个独立的专家子网络，每个 token 通过路由机制只激活其中一部分专家。

细粒度专家路由：DeepSeek V4-Pro 将传统 FFN 层拆分为 256+ 个细粒度专家（Fine-grained Experts），每个专家的维度仅为标准 FFN 维度的 1/8 到 1/16。对于每个输入 token，路由网络（Router）会计算所有专家的得分，选择 Top-8 到 Top-12 个最相关的专家进行激活。这种设计使专家分工更加明确——有的专家擅长数学推理，有的擅长代码生成，有的擅长长文本理解。

共享专家机制：除了路由专家外，V4-Pro 还引入了 1-2 个始终激活的共享专家（Shared Expert）。共享专家负责处理所有 token 共有的基础特征变换，如语法解析、基础语义编码等。这种设计的灵感来源于观察：在自然语言处理中，大部分 token 需要经历相似的底层语义变换，将这些共性计算从路由专家中剥离出来，可以避免冗余计算，提升参数利用效率。

负载均衡策略：MoE 架构面临的一个关键挑战是专家坍缩（Expert Collapse）——某些专家可能被过度使用，而其他专家几乎不被激活。V4-Pro 通过三种策略解决这一问题：辅助损失（Auxiliary Loss）在训练时惩罚负载不均衡；动态调整路由权重，根据历史使用情况重新分配；Expert Drop 机制在负载过高时主动丢弃部分 token 的专家激活。

| 对比维度 | 传统 MoE（如 Mixtral 8x7B） | DeepSeek-MoE V2（V4-Pro） |
| --- | --- | --- |
| 专家总数 | 8-16 个粗粒度专家 | 256+ 细粒度专家 |
| 每个专家维度 | 与 FFN 层维度一致 | FFN 维度的 1/8~1/16 |
| 激活专家数 | 2-4 个（Top-2~4） | 8-12 个（Top-8~12） |
| 共享专家 | 无 | 1-2 个共享专家 |
| 负载均衡 | 辅助损失 + 容量因子 | 动态调整 + Expert Drop |
| 性能提升 | 基线 | +5-15% |

### 6.1.3 MLA 注意力机制

MLA（Multi-head Latent Attention）是 DeepSeek 自研的注意力机制创新，也是 V4-Pro 在推理效率方面的核心竞争力。传统的 MHA（Multi-Head Attention）需要为每个注意力头和每个 token 存储完整的 Key 和 Value 矩阵，导致 KV Cache 随序列长度线性增长，成为长上下文推理的主要瓶颈。

MLA 的核心思路是将 Key 和 Value 投影到低秩潜在空间（Latent Space）。具体来说，MLA 不再直接存储高维的 K 和 V 张量，而是存储一个低维的潜在向量（Latent Vector），在计算注意力分数时再通过上投影矩阵恢复为完整的 K 和 V。这个过程可以形象地理解为压缩-解压缩机制：存储时压缩到低维空间以节省显存，计算时解压以保持精度。

| 对比维度 | MHA（标准多头注意力） | GQA（分组查询注意力） | MLA（多头潜在注意力） |
| --- | --- | --- | --- |
| KV 头数 | 等于 Q 头数 (H) | H / G（分组） | 1（潜在空间） |
| KV Cache/头 | 2 x d_h x L | 2 x d_h x L | 2 x d_lv x L（低秩） |
| KV Cache 降幅 | 基线 | 约 4x（G=8） | 10x-15x |
| 推理精度影响 | 基准 | 几乎无损 | 几乎无损 |
| 实现复杂度 | 低 | 中 | 高（需上下投影矩阵） |
| 代表模型 | LLaMA 1 | LLaMA 2/3 / Qwen 3.7 | DeepSeek V2/V3/V4 |

MLA 的实际效果非常显著：以 128K 上下文长度为例，DeepSeek V4-Pro 的 KV Cache 占用约为同等规模的 MHA 模型的 1/10 到 1/15。这意味着在相同的 GPU 显存预算下，V4-Pro 可以支持更长的上下文或更大的 batch size，从而提升推理吞吐。当然，MLA 也有代价：需要额外的上下投影矩阵，在训练时增加了一定的复杂度，但在推理时的收益远大于这些开销。

### 6.1.4 网络结构特点

除了上述 MoE 和 MLA 两大核心创新外，DeepSeek V4-Pro 在网络结构的其他方面也采用了当前业界最优的设计选择，共同构成了其强大的推理能力。

FFN 激活函数 - SwiGLU：V4-Pro 的 FFN 层使用 SwiGLU（Swish-Gated Linear Unit）激活函数，其数学形式为 FFN(x) = (Swish(xW1) ⊙ (xW2)) W3。与传统的 ReLU 相比，SwiGLU 通过门控机制引入了非线性特征选择的灵活性，可以更精确地控制信息的流动，在多项 benchmark 上带来约 2-5% 的性能提升。

归一化层 - RMSNorm：V4-Pro 使用 RMSNorm（Root Mean Square Normalization）替代 LayerNorm。RMSNorm 省去了 LayerNorm 中的均值计算，只保留方差归一化，计算量减少约 15%。在深层 Transformer 中，这种计算节省随着层数增加而累积，对于 60-80 层的模型来说效果显著。

位置编码 - RoPE：采用旋转位置编码（Rotary Position Embedding），通过在注意力计算中引入旋转矩阵，将相对位置信息编码到 Q 和 K 向量的点积中。RoPE 天然支持相对位置感知，使模型可以更好地泛化到训练时未见过的序列长度，支持上下文外推至 128K-1M。

Multi-Token Prediction (MTP)：V4-Pro 支持同时预测未来多个 Token 的训练策略。在推理时，MTP 可以作为一种加速模式使用——模型一次生成多个候选 token，然后通过验证器批量验证，从而提升推理吞吐。这是 DeepSeek 在训练-推理协同优化方面的重要创新。

## 6.2 Qwen 3.7 架构分析

### 6.2.1 总体架构概览（输入层→隐藏层→输出层）

Qwen 3.7 是阿里巴巴通义千问团队于 2025 年推出的大语言模型系列重要版本。与 DeepSeek 的 MoE 路线不同，Qwen 3.7 坚持 Dense Transformer 密集型架构，不依赖专家路由，全参数激活。以下从输入层、隐藏层、输出层三个维度拆解其完整网络结构。

| 网络层次 | 组件名称 | 具体配置 | 功能说明 |
| --- | --- | --- | --- |
| 输入层 | Token Embedding | 词表 ~152K，嵌入维度 7168（72B） | 多语言优化 BPE，将 Token ID 映射为稠密向量 |
| 输入层 | Position Encoding | RoPE + NTK-aware + YaRN | 多层扩展策略，支持 32K-256K 上下文 |
| 隐藏层 | Transformer Block × N | 约 80 层（72B 版本）Decoder-Only | Pre-RMSNorm → GQA Attention → Residual → Pre-RMSNorm → FFN(SwiGLU) → Residual |
| 隐藏层 | GQA Attention | Q 头 H=64, G=8, K/V 头=8 | 分组查询注意力，KV Cache 降 75% |
| 隐藏层 | FFN SwiGLU | d_ff ≈ 28672（4x 扩展比） | 门控 FFN，三组权重矩阵 W1/W2/W3 |
| 隐藏层 | Pre-RMSNorm | 每子层前归一化 | 计算量少于 LayerNorm，训练更稳定 |
| 隐藏层 | Residual 残差连接 | Pre-Norm: x + Sublayer(Norm(x)) | 标准 Transformer 设计 |
| 输出层 | Final RMSNorm | 最后一层归一化 | 稳定输出分布 |
| 输出层 | LM Head | d_model → 词表 152K | 线性投影输出 logits，可选权重绑定 |

Qwen 3.7 的完整数据流为：输入 Token → Embedding + RoPE → N 层 Transformer Block（每层执行 GQA 注意力 + SwiGLU FFN）→ Final RMSNorm → LM Head → 输出 logits。与 DeepSeek V4-Pro 相比，Qwen 3.7 的架构更加简洁——没有 MoE 路由、没有低秩压缩，每层 FFN 都是完整的全连接网络。

### 6.2.2 GQA 分组查询注意力

Qwen 3.7 采用 GQA（Grouped Query Attention）作为其注意力机制。GQA 是介于 MHA（多头注意力）和 MQA（多查询注意力）之间的折中方案，在推理效率和模型精度之间取得了良好的平衡。

GQA 的核心原理是将 Q（查询）头分组，每组内的 Q 头共享同一组 K（键）和 V（值）头。Qwen 3.7 在 72B 版本中采用 G=8 的分组设置，即 Q 头数 H=64，K/V 头数 = H/G = 8。这意味着每 8 个 Q 头共享一组 KV 头，相比 MHA 的 64 组 KV 头，KV Cache 减少了 75%。

| 对比维度 | MHA（多头注意力） | MQA（多查询注意力） | GQA（分组查询注意力） |
| --- | --- | --- | --- |
| KV 头数 | 等于 Q 头数 (H) | 1 | H/G（G=8 时 8 个） |
| KV Cache 降幅 | 基线 | H 倍 | G 倍（约 75%） |
| 精度损失 | 无 | 有损失 | 几乎无损 |
| 实现复杂度 | 低 | 低 | 低 |
| 代表模型 | LLaMA 1 | PaLM | LLaMA 2/3, Qwen 3.7 |

GQA 的优势在于实现简洁，不改变注意力计算的核心逻辑，仅减少 KV 头的数量。这使得 Qwen 3.7 在推理框架中的兼容性非常好——无论是 vLLM、SGLang 还是 llama.cpp，都对 GQA 有原生支持，无需特殊优化。

### 6.2.3 SwiGLU 激活与 FFN 结构

Qwen 3.7 的 FFN 层使用 SwiGLU 激活函数，这是当前大语言模型中最主流的 FFN 设计之一。SwiGLU 的数学形式为：

FFN(x) = (Swish(xW1) ⊙ (xW2)) W3

其中 ⊙ 表示逐元素乘法，Swish(x) = x · sigmoid(x) 是平滑版的 ReLU。与传统的 ReLU FFN 相比，SwiGLU 通过门控机制引入了更灵活的非线性特征选择能力。与 GeLU FFN 相比，SwiGLU 在多项标准 benchmark 上可以带来约 2%-5% 的性能提升。

参数代价：SwiGLU 的代价是引入了三组权重矩阵（W1, W2, W3）而非标准的 ReLU FFN 的两组（W1, W2）。因此，FFN 层的参数量和计算量增加了约 33%。在 Qwen 3.7 的设计中，这种参数预算的分配被认为是值得的——FFN 层的质量提升直接影响模型的推理能力和知识存储密度。

与 MoE 的 FFN 差异：Qwen 3.7 的 FFN 是标准的全连接层，所有参数在每次推理时都被激活。这与 DeepSeek V4-Pro 的 MoE FFN 形成鲜明对比：Qwen 的每层 FFN 只有一个专家，但参数利用率更高（100% 激活）；DeepSeek 的每层 FFN 有数百个专家，但每次只激活 3-5%。

### 6.2.4 长上下文与 RoPE 优化

Qwen 3.7 在长上下文能力方面做出了多项关键优化，使其在 32K-256K 的上下文范围内都能保持稳定的推理质量。这些优化主要围绕 RoPE 位置编码展开。

| 优化技术 | 核心原理 | Qwen 3.7 实现 | 效果 |
| --- | --- | --- | --- |
| RoPE base 频率调优 | 调整旋转频率基频 | base=10,000 → 500,000 | 支持 32K → 128K 直接外推 |
| NTK-aware 缩放 | 非均匀缩放旋转频率 | 动态 α=2-8 | 保留短上下文精度 |
| YaRN 扩展 | 改进 NTK 的缩放方案 | 集成到训练前扩展 | 256K 稳定推理 |
| LogN 注意力缩放 | 按序列长度缩放 logits | 自动适配长序列 | 防止注意力弥散 |

这些优化技术的组合使用，使得 Qwen 3.7 可以在不经过长上下文微调的情况下，直接外推到训练时未见过的更长序列长度。例如，训练时最大长度为 32K 的模型，通过 RoPE base 频率调优和 NTK-aware 缩放，可以直接推理 128K 的输入，而无需额外的微调步骤。

## 6.3 架构对比分析

### 6.3.1 相同点

尽管 DeepSeek V4-Pro 和 Qwen 3.7 在架构路线上有显著差异（MoE vs Dense），但两者在底层设计上存在许多共同点，这些共同点反映了当前大语言模型架构设计的主流共识。

| 共同设计 | 说明 | 技术意义 |
| --- | --- | --- |
| 基础架构同源 | 均采用 Decoder-Only Transformer | 自回归生成模型的标准范式 |
| 归一化策略一致 | 均使用 Pre-RMSNorm | 省去均值计算，减少约 15% 开销 |
| 位置编码一致 | 均采用 RoPE 旋转位置编码 | 天然支持相对位置感知和上下文外推 |
| 激活函数一致 | FFN 均使用 SwiGLU 门控激活 | 提升特征选择能力，约 2-5% 性能提升 |
| 残差结构一致 | 均采用 Pre-Norm 残差连接 | 确保深层网络训练稳定性 |
| 输出层设计一致 | Final RMSNorm + LM Head | 支持权重绑定，稳定输出分布 |
| 训练范式一致 | 预训练 + RLHF/GRPO 对齐 | 大规模数据 + 强化学习对齐 |

这些共同点表明，大语言模型的架构设计已经趋于收敛——Decoder-Only、Pre-RMSNorm、RoPE、SwiGLU 已经成为业界共识的标准配置。两者的差异主要体现在如何组织 FFN 层和如何优化注意力这两个核心问题上。

### 6.3.2 不同点

DeepSeek V4-Pro 和 Qwen 3.7 在 6 个关键维度上存在显著差异，这些差异反映了两种不同的设计哲学和工程取舍。

6.3.2.1 架构路线差异：DeepSeek V4-Pro 采用 MoE 稀疏激活架构，总参数 ~1T，激活参数 ~250B，用稀疏性换取模型容量。Qwen 3.7 采用 Dense 密集型架构，全参数激活，无路由开销，用简洁性换取部署效率。MoE 的优势在于参数容量大、推理吞吐高；Dense 的优势在于训练稳定、部署简单。
6.3.2.2 注意力机制差异：DeepSeek V4-Pro 使用 MLA（多头潜在注意力），通过低秩压缩将 KV Cache 降低 10-15x，是超长上下文场景的核心优势。Qwen 3.7 使用 GQA（分组查询注意力），通过分组共享将 KV Cache 降低 75%，兼顾实现简洁和效率提升。
6.3.2.3 词表与嵌入维度差异：DeepSeek V4-Pro 词表约 128K，侧重中英文通用场景。Qwen 3.7 词表约 152K，采用多语言优化的 BPE 算法，多语言覆盖更广，对非英语语种的支持更好。
6.3.2.4 上下文长度差异：DeepSeek V4-Pro 原生支持 128K-1M tokens，MLA 的低秩压缩使长上下文下的显存占用可控。Qwen 3.7 支持 32K-256K tokens，通过 RoPE 调优和 NTK/YaRN 扩展实现，实现方式更灵活但显存效率不如 MLA。
6.3.2.5 参数规模策略差异：DeepSeek V4-Pro 采用单一超大模型策略（~1T），通过 MoE 稀疏激活在不同场景下自适应。Qwen 3.7 采用多尺寸全系列策略（7B-110B），提供多个尺寸版本供用户按需选择，从小型端侧模型到大型云端模型全覆盖。
6.3.2.6 推理特性差异：DeepSeek V4-Pro 支持 Multi-Token Prediction (MTP) 推理加速，MoE 路由调度可针对不同场景优化。Qwen 3.7 支持 Thinking Mode（深度思考模式）与 Normal Mode（快速响应模式）双模式切换，灵活适配不同推理需求。
### 6.3.3 各自优势

两种架构路线各有独特的优势，适用于不同的应用场景。

6.3.3.1 DeepSeek V4-Pro 优势：
1. 参数容量优势：总参数 ~1T，知识覆盖面远超同激活参数量的 Dense 模型，在处理复杂推理任务时表现更优。
2. 推理吞吐优势：稀疏激活仅用 ~250B 参数，推理吞吐为同级别 Dense 模型的 2-3 倍，在高并发场景下性价比更高。
3. 长上下文优势：MLA 机制使 KV Cache 降低 10-15x，128K 上下文显存仅为 MHA 的 1/10，支持 1M Token 超长序列。
4. 成本优势：同等模型能力下，推理成本仅为 Dense 模型的 1/2 到 1/3，适合大规模部署。
5. MTP 优势：Multi-Token Prediction 可在推理时批量验证，加速生成过程。
6.3.3.2 Qwen 3.7 优势：
1. 部署简单优势：Dense 架构无路由负载均衡问题，开箱即用，框架兼容性最佳。
2. 系列化优势：7B-110B 多尺寸覆盖，端侧到云端一套架构统一部署，按需选择最适配的模型尺寸。
3. 训练稳定优势：Dense 架构训练无专家坍缩风险，微调更稳定可靠，对数据质量要求相对较低。
4. 生态成熟优势：HuggingFace / vLLM / SGLang / llama.cpp 等框架原生支持，无需特殊配置，社区资源丰富。
5. 双模式推理优势：Thinking Mode 深度推理 + Normal Mode 快速响应，灵活适配不同场景。
### 6.3.4 选型建议

在实际推理系统的架构选型中，建议根据具体场景和需求选择最合适的模型：

| 应用场景 | 推荐模型 | 选型理由 |
| --- | --- | --- |
| 高并发在线 API 服务 | DeepSeek V4-Pro | MoE 高吞吐，推理成本低，性价比最优 |
| 企业私有化部署 | Qwen 3.7（72B） | Dense 更易运维，框架兼容性最好 |
| 超长文档理解 (100K+) | DeepSeek V4-Pro | MLA 长上下文显存优势明显 |
| 端侧/边缘部署 | Qwen 3.7（7B/14B） | Dense 部署简单，生态成熟，兼容性好 |
| 代码/数学推理 | DeepSeek V4-Pro | MoE 大容量参数覆盖更多知识模式 |
| 研究/微调场景 | Qwen 3.7 | Dense 微调更稳定，社区资源丰富 |
| 多语言场景 | Qwen 3.7 | 词表 152K，多语言优化 BPE 覆盖更广 |

总体来看，DeepSeek V4-Pro 和 Qwen 3.7 并非简单的尤尤得犹忪关系，而是在不同的设计约束下做出了不同的架构取舍。对于推理系统工程师而言，理解这两种架构路线的核心差异，有助于在实际项目中做出更优的模型选型和部署决策。
